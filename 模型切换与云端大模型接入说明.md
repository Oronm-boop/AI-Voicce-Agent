# 模型配置说明

以后只改根目录这个文件：

```text
model_config.json
```

后端启动、聊天调用、模型状态检查、设置页保存，都会使用这一份配置。`local-agent/.env` 只作为兜底默认值，不再推荐改。

## 配置文件

```json
{
  "llm_provider": "ollama",
  "llm_base_url": "http://127.0.0.1:11434",
  "llm_model": "qwen2.5:0.5b",
  "allow_remote_llm": false,
  "llm_api_key": "",
  "llm_api_key_env": "",
  "enable_thinking": false,
  "default_max_tokens": 2048,
  "request_timeout_seconds": 120
}
```

## 本地 Ollama

```json
{
  "llm_provider": "ollama",
  "llm_base_url": "http://127.0.0.1:11434",
  "llm_model": "qwen2.5:7b",
  "allow_remote_llm": false
}
```

## 本地 OpenAI-compatible 服务

例如 LM Studio、vLLM、LocalAI、Ollama `/v1`：

```json
{
  "llm_provider": "openai_compatible",
  "llm_base_url": "http://127.0.0.1:1234/v1",
  "llm_model": "模型服务返回的模型 id",
  "allow_remote_llm": false
}
```

## 云端 OpenAI-compatible 服务

```json
{
  "llm_provider": "openai_compatible",
  "llm_base_url": "https://你的供应商地址/v1",
  "llm_model": "云端模型名",
  "allow_remote_llm": true,
  "llm_api_key": "你的 API Key"
}
```

如果不想把 Key 写进文件，也可以这样：

```json
{
  "llm_provider": "openai_compatible",
  "llm_base_url": "https://你的供应商地址/v1",
  "llm_model": "云端模型名",
  "allow_remote_llm": true,
  "llm_api_key_env": "MY_LLM_API_KEY"
}
```

然后在系统环境变量里设置 `MY_LLM_API_KEY`。

## 重要规则

- 改完 `model_config.json` 后，重启 `local-agent` 后端。
- `llm_base_url` 以 `/v1` 结尾时，会走 OpenAI-compatible `/chat/completions`。
- `llm_provider` 推荐只填 `ollama` 或 `openai_compatible`。
- `allow_remote_llm=false` 时，只允许 `127.0.0.1`、`localhost`、`::1`。
- `model_config.json` 已经加入 `.gitignore`，不要把真实 API Key 提交到仓库。
