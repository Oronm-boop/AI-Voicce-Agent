# 下一代智能体需求文档（实时自然语音 + 任务 Agent）

# 开发PC桌面端软件，完全调用本地部署的大模型

## 1. 定义

实时自然语音智能体，是一种能像真人通话一样持续语音交流，并将语音意图实时转化为理解、回应和任务执行的 AI Agent。

## 2. 核心能力：任务闭环

这是它区别于单纯语音助手的本质：它不仅告诉你“怎么做”，而是直接“替你做”。

- **自主拆解目标**：当你下达一个模糊指令（如“帮我策划下周去大理的旅行，订好性价比最高的机票和带露台的酒店”），它会自主进行任务规划。
- **工具调用（Tool Use）**：它能自主登录各类 App、调用 API、操作浏览器，甚至在电脑/手机系统层进行点击和输入。
- **决策与确认**：在关键步骤（如支付、删除敏感文件）时，它会通过语音确认，例如：“机票已选好，总价 1200 元，需要我现在下单吗？”

## 3. 典型使用场景画像

### 场景描述

你戴着耳机在开车，随口说：

> 帮我查一下刚才那封关于项目进度的邮件，里面提到的几个技术难点，你结合我最近写的 Rust 代码，给我写一个重构方案，发到 Slack 频道里。

### 智能体执行链路（听-想-做-回）

- **听（实时语音）**：捕捉指令，并从语气中识别你当前很匆忙。
- **想（任务规划）**：调取 Gmail API 搜索邮件，识别附件内容。
- **做（任务执行）**：访问 GitHub 仓库（或本地 WSL2 环境），分析 Rust 代码逻辑。
- **回（结果反馈）**：以简练语音返回结果，例如：“重构方案已生成，核心是解决了所有权的生命周期问题，已发送至 Slack，你可以等停下车后再查看细节。”

## 4. 什么是“实时自然语音”

“实时自然语音”不是简单的：

> 按一下语音按钮 → 录音 → 转文字 → 发给大模型 → 再播报结果

它更像是：

> 用户一开口，系统就持续听、持续理解、持续判断意图，并能边听边回应、边确认、边执行任务。

## 5. 与传统语音交互的差异

### 传统语音流程

按住/点击语音按钮 → 录完 → ASR 转文字 → 文本发给模型 → 模型生成回复 → TTS 播放语音

### 实时自然语音流程

持续语音输入 → 实时语音理解 → 模型即时推理 → 自然语音回应/追问/执行任务

### 关键差异点

1. **无需反复点击语音按钮**  
   可像通话一样保持会话状态，用户直接说话即可。
2. **不必等用户完全说完再处理**  
   可边听边理解，识别停顿、未说完、改口等语音行为。
3. **支持自然打断**  
   用户在系统回答中说“等等，不是这个意思”，系统应能立即停止并调整。
4. **语音不止是“转文字”**  
   系统可理解语气、节奏、情绪、犹豫、强调等副语言信息；例如语气急促可能意味着任务优先级更高。
5. **可直接进入任务执行**  
   例如用户说“帮我给张伟发个邮件，说会议改到明天下午三点”，系统应理解为完整任务链：找联系人、生成邮件、确认内容、创建草稿或发送。

## 6. 补充说明

底层通常仍包含语音识别、语音理解、语音合成等模块，但在用户体验上不再是“录音转文字发消息”，而是更接近实时对话式智能体。
